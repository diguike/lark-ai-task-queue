# 启动提示词:把 Lark AI Task Queue 实现出来

> 复制下面这段,在 `lark-ai-task-queue/` 目录里新开一个 Claude Code 会话发给它。
> 这段提示词负责把脚手架变成真正可跑的闭环。

---

我在 `lark-ai-task-queue/` 目录下有一个项目脚手架,目标是:**把飞书的一个专属任务清单当作"寄给 AI 的待办队列",AI 定时拉取里面未完成的任务、自动执行、产出飞书文档,完成后把文档链接回写到任务评论并标记完成。**

请先读 `README.md`、`config/config.example.json`、`prompts/run-queue.md` 理解设计,然后帮我把功能实现出来。具体要做:

1. **摸清飞书任务的能力**:用 `lark-task` skill 探索 API —— 如何列出某个任务清单(tasklist)下的未完成任务、如何给任务加评论、如何标记任务完成、如何按 guid 定位清单。用 `lark-doc` skill 确认如何创建飞书文档并拿到分享链接。把关键命令记录到 README 或一个 `NOTES.md`。

2. **建清单 + 落配置**:帮我在飞书创建(或让我指定一个已有的)任务清单作为 "AI 队列",查出它的 guid,生成 `config/config.json`(从 example 复制并填好 `tasklist_guid`)。

3. **实现执行逻辑**:按 `prompts/run-queue.md` 的步骤,做成可以"跑一轮"的形式 —— 拉取未完成任务 → 逐条执行 → 建飞书文档 → 回写评论 → 标记完成 → 写 `logs/YYYY-MM-DD.log`。先保证手动跑一轮(end-to-end)能通。

4. **接上定时**:跑通后,告诉我用 `/loop` 定时触发的确切命令,以及一个可选的 `cron + claude -p` headless 方案(脚本放 `scripts/`)。

5. **端到端验证**:让我先在 "AI 队列" 里放一条测试任务(比如"调研 X 并写一页总结"),你完整跑一遍,把飞书文档链接和回写结果给我看。

要求:
- 高风险动作(删除、外发、花钱)默认先停下来等我确认,别自动执行。
- 单条任务失败不要中断整轮,记日志 + 在任务下评论错误。
- 只动 "AI 队列" 这一个清单,别碰我其他飞书任务。
- 实现过程中如果发现 `README` 里的设计有更优解,先告诉我再改。

先从第 1 步开始,把飞书 API 摸清楚后给我一个简短的实现计划,我确认后再动手。
